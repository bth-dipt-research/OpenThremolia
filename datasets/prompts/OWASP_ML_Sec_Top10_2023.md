## OWASP ML Security Top 10 2023:
ML01:2023 Input Manipulation Attack - Is an umbrella term, which include Adversarial Attacks, a type of attack in which an attacker deliberately alters input data to mislead the model.
ML02:2023 Data Poisoning Attack - Occur when an attacker manipulates the training data to cause the model to behave in an undesirable way.
ML03:2023 Model Inversion Attack - Occur when an attacker reverse-engineers the model to extract information from it.
ML04:2023 Membership Inference Attack - Occur when an attacker manipulates the model's training data in order to cause it to behave in a way that exposes sensitive information.
ML05:2023 Model Theft - Occur when an attacker gains access to the model's parameters.
ML06:2023 AI Supply Chain Attacks - Threat actors target the supply chain of ML models. It consists of specific elements such as MLOps platforms, data management platforms, model management software, model hubs and other specialized types of software that enable ML engineers to effectively test and deploy software.
ML07:2023 Transfer Learning Attack - Occur when an attacker trains a model on one task and then fine-tunes it on another task to cause it to behave in an undesirable way.
ML08:2023 Model Skewing - Occur when an attacker manipulates the distribution of the training data to cause the model to behave in an undesirable way.
ML09:2023 Output Integrity Attack - An attacker aims to modify or manipulate the output of a machine learning model in order to change its behavior or cause harm to the system it is used in.
ML10:2023 Model Poisoning - Occur when an attacker manipulates the model's parameters to cause it to behave in an undesirable way.
